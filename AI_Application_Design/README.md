# AI Application Design

Resources and references for architecting AI applications — from planning through production.

## Documents

- **[SDLC.md](./SDLC.md)** — Describes the 10 SDLC phases adapted for AI projects, from feasibility assessment to governance. Each phase covers AI-specific activities, key artifacts, and a comparison with traditional SDLC to highlight where AI development diverges from conventional software engineering.

- **[Tools_To_Consider.md](./Tools_To_Consider.md)** — A curated reference of popular tools and frameworks mapped to each AI SDLC phase. Covers everything from data preparation (DVC, Great Expectations) and model development (LangChain, MLflow, Unsloth) to deployment (BentoML, Triton), monitoring (WhyLabs, LangFuse), and governance (OPA, Purview).

- **[AI_Tool_Selection_Strategy.md](./AI_Tool_Selection_Strategy.md)** — A decision framework for selecting AI tools, models, and platforms across Gen AI and Traditional ML projects. Covers build-vs-buy analysis, model selection criteria, vendor evaluation matrix, TCO modeling, risk assessment, and quick-reference decision tables for common use cases.

- **[Tool_Shortlisting_Workflow.md](./Tool_Shortlisting_Workflow.md)** — A repeatable stage-gated process for organizations to shortlist AI tools. Walks through 5 stages (Requirements → Knockout Filtering → Weighted Scoring → POC → Final Sign-off) with iteration loops, timeline estimates for 5 organizational contexts, and ready-to-use templates (requirements brief, longlist tracker, scoring matrix, POC report, decision record).
